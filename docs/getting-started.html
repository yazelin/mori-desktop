Getting Started

INSTALL · DEV · FIRST RUN

從 0 到 Mori 在你 Ubuntu 上跑起來 — 三步驟。

主力平台:Ubuntu 26.04 + GNOME Wayland。 Windows / macOS 的 paste-back / 全域熱鍵尚未接,主視窗 UI 跑得起來但語音輸入流程不完整。 歡迎 fork + PR 幫忙補 — 見 GitHub Issues。

1. 系統需求

  • Rust 1.94+
  • Node 22+
  • (Linux)Tauri 2 build deps + wl-clipboard + ydotool

Ubuntu 26.04 一條龍 setup script: yazelin/ubuntu-26.04-setup 

./setup-rust.sh           # Rust toolchain
./setup-tauri-deps.sh     # WebKitGTK + ALSA + tray libs
./setup-wayland-input.sh  # wl-clipboard + ydotool daemon

裝完要重開機一次 讓 input group 生效(ydotool 需要)。

2. Clone + Build

git clone https://github.com/yazelin/mori-desktop.git
cd mori-desktop

cargo build --workspace   # workspace 才會 build mori-cli (Bash CLI proxy 需要)
npm install

3. 跑 Dev 模式

npm run tauri dev

第一次 build 在 Intel Ultra 上 ~3 分鐘。之後 incremental ~10 秒。

注意:npm run tauri dev 只 build mori-tauri。 mori CLI(Bash CLI proxy 用)要另外跑 cargo build -p mori-cli, binary 在 target/debug/mori。

主畫面截圖

主視窗 = sidebar shell(6 個 tab)+ 桌面常駐 floating sprite + Ctrl+Alt+P picker overlay。

第一次啟動會發生的事

  1. GNOME 跳權限對話框:問你要不要讓 Mori 註冊全域熱鍵 — 點「新增(Add)」。 誤點拒絕的修法見 Troubleshooting → 全域熱鍵
  2. ~/.mori/ 建立:config.json stub / themes/dark.json / themes/light.json / 空的 voice_input / agent 目錄
  3. 主視窗顯示:sidebar shell + 桌面右下 floating sprite(160×160)
  4. 第一次按 Ctrl+Alt+Space:錄音 → STT → 但會抱怨「沒設 LLM provider」
    → 去 Config tab 填 Groq key(預設 provider)或選離線組合 — 見 Providers
